# 기획/분석 파이프라인 템플릿

## 5단계 구체적 행동

| 단계 | 구체적 행동 | 출력물 | 사용 도구/방법론 |
|------|-----------|--------|----------------|
| Input | 의사결정 배경 파악, 분석 범위 정의, 데이터 소스 확인 | CPS + 분석 스코프 정의서 | AskUserQuestion, CPS 프레임 |
| Plan | 분석 프레임워크 선정, 가설 수립, 비교 축 설정 | 분석 프레임 + 가설 목록 | TodoWrite, MECE/SWOT/Porter 등 |
| Design | 보고서 구조, 시각화 유형, 결론 도출 기준 명세 | 분석 Blueprint | SKILL.md Blueprint |
| Execute | 데이터 수집·가공, 프레임워크 적용, 인사이트 도출 | 분석 결과 + 시각화 | xlsx 스킬, Python, 차트 라이브러리 |
| Lint | 논리적 일관성, 데이터 출처 명시, MECE 검증, 결론-근거 정합 | 검증 보고서 + 최종 분석서 | 분석 Lint Rules (A-01~A-05) |

## Input 단계 상세

AskUserQuestion으로 수집할 항목:

1. **Context**: 이 분석이 필요한 배경은? 어떤 의사결정을 지원하는가?
2. **Problem**: 현재 정보 공백은? 무엇을 알아야 결정할 수 있는가?
3. **Solution**: 보고서 형식, 시각화 요구, 의사결정 기한
4. **데이터 소스**: 활용 가능한 데이터, 접근 경로, 신뢰도

## Plan 단계 상세

분석 유형에 따라 프레임워크를 선정한다:

| 분석 유형 | 적합한 프레임워크 |
|-----------|-----------------|
| 시장 분석 | Porter's Five Forces, PESTLE |
| 경쟁 분석 | SWOT, 벤치마킹 매트릭스 |
| 전략 수립 | Ansoff Matrix, BCG Matrix |
| 의사결정 | 의사결정 매트릭스, 비용-편익 분석 |
| 문제 진단 | 이시카와 다이어그램, 5 Whys |

가설을 먼저 수립하고, 각 가설을 검증할 데이터와 방법을 매핑한다.

## Design 단계 상세

Blueprint에 포함할 항목:

- **보고서 구조**: Executive Summary → 분석 본문 → 결론 → 제안
- **시각화 계획**: 각 섹션에서 사용할 차트 유형 (막대, 라인, 파이 등)
- **결론 도출 기준**: "A가 B보다 30% 이상이면 X를 선택" 같은 명시적 기준
- **분량**: 각 섹션별 페이지/분량 배분

## Execute 단계 위임

- 데이터 분석이 필요한 경우 → `xlsx` 스킬 + Python 스크립트
- 보고서 작성 → `analysis-harness` 스킬 + `docx`/`pptx` 스킬
- 시각화 → Recharts, Plotly, 또는 xlsx 차트

## Lint 규칙

| ID | 규칙 | 검증 방법 |
|----|------|----------|
| A-01 | 모든 주장에 데이터 출처가 명시되어 있는가 | 주장-출처 1:1 매핑 확인 |
| A-02 | 분석 프레임워크가 MECE를 충족하는가 | 범주 간 중복/누락 검증 |
| A-03 | 결론이 데이터로부터 논리적으로 도출 가능한가 | 결론 ← 근거 역추적 |
| A-04 | 가설과 검증 결과가 1:1로 매핑되는가 | 가설 목록 vs 검증 결과 대조 |
| A-05 | 의사결정자를 위한 실행 가능한 제안(Actionable)이 포함되어 있는가 | 제안 섹션 존재 + 구체성 확인 |

### A-01 상세: 출처 명시

모든 데이터 기반 주장에는 다음 중 하나가 반드시 동반되어야 한다:
- 데이터 파일명 + 시트/컬럼 참조
- 외부 출처 URL + 발행일
- 인터뷰/설문 결과의 경우 출처 명시

### A-02 상세: MECE 검증

선택한 분석 프레임워크의 범주들이:
- 상호 배타적(Mutually Exclusive)인가: 하나의 항목이 두 범주에 동시에 속하지 않는가
- 전체 포괄적(Collectively Exhaustive)인가: 분석 대상의 모든 측면이 커버되는가

### A-03 상세: 논리적 도출 검증

결론에서 역으로 추적하여:
- 결론을 지지하는 근거가 최소 2개 이상 존재하는가
- 근거가 결론과 직접적 인과관계 또는 상관관계에 있는가
- 대안적 해석의 가능성이 언급되었는가

### A-05 상세: Actionable 제안

제안이 다음 조건을 충족하는지 확인:
- Who: 누가 실행하는가
- What: 구체적으로 무엇을 하는가
- When: 언제까지 실행하는가
- How: 실행 방법이 구체적인가

## 파이프라인 인스턴스 예시

```
📌 업무 유형: 기획/분석
📌 이해관계자: 경영진(의사결정자) / 마케팅팀 / 재무팀

━━━ 자동 생성된 파이프라인 ━━━

[Input] AskUserQuestion으로 수집:
  - Context: 신규 시장 진출 검토를 위한 시장 분석
  - Problem: 어떤 시장이 가장 높은 ROI를 줄 수 있는가?
  - Solution: 경영진 보고용 10페이지 분석 보고서
  - 데이터: 내부 매출 데이터 + 시장 리서치 자료

[Plan] 분석 전략:
  1. 프레임워크: Porter's Five Forces + 시장 규모 분석
  2. 가설: "동남아 시장이 가장 높은 성장 잠재력"
  3. 비교 축: 시장 규모, 경쟁 강도, 진입 장벽, 규제 환경
  4. 데이터 소스: Statista, 내부 매출 DB, 현지 파트너 인터뷰

[Design] Blueprint:
  - 구조: Executive Summary → 시장 개관 → 5 Forces 분석 → 비교 매트릭스 → 결론 → 제안
  - 시각화: 시장 규모 막대 차트, 경쟁 강도 레이더 차트, 비교 매트릭스 표
  - 결론 기준: 종합 점수 상위 2개 시장 선정

[Execute] analysis-harness → xlsx → pptx 스킬 체인

[Lint] 검증 체크리스트:
  □ A-01: 모든 데이터에 출처 명시
  □ A-02: 5 Forces 분석이 MECE 충족
  □ A-03: 결론이 데이터에서 논리적 도출
  □ A-04: 가설-검증 1:1 매핑 완료
  □ A-05: 경영진을 위한 Actionable 제안 포함
```
