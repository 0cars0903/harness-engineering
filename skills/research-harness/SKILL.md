# Research Harness — 리서치 전용 하네스

리서치 작업에서 출처 신뢰성, 관점 균형, 답변 완결성을 보장하는 하네스다.

---

## 워크플로우

### Step 1: 리서치 스코핑

다음을 확인한다:

1. **Research Questions (RQ)**: 구체적 리서치 질문 목록
2. **깊이 수준**: 개관(Survey) / 심층(Deep Dive) / 전문가급(Expert)
3. **기간 범위**: 언제부터 언제까지의 정보를 대상으로 하는가
4. **소스 제한**: 특정 소스만 사용하는가, 제외할 소스가 있는가
5. **결과물 형식**: 요약/보고서/비교표/프레젠테이션

깊이 수준별 기대치:

| 수준 | 소스 수 | 분량 | 분석 깊이 |
|------|---------|------|----------|
| 개관 | 3~5개 | 1~2페이지 | 주요 사실 정리 |
| 심층 | 10~15개 | 5~10페이지 | 다각도 비교 분석 |
| 전문가급 | 20개+ | 10페이지+ | 메타 분석, 원본 데이터 검토 |

### Step 2: 리서치 계획 수립

TodoWrite로 리서치 계획을 작성한다:

1. **소스 전략**: 어디서, 어떤 키워드로 검색하는가
   - 1차 소스: 공식 문서, 원 데이터, 학술 논문
   - 2차 소스: 분석 기사, 전문가 블로그, 산업 보고서
   - 3차 소스: 위키, 교과서, 종합 가이드
2. **키워드 매트릭스**: RQ별 검색 키워드 조합
3. **수집 템플릿**: 소스별 기록 항목 (출처, 날짜, 핵심 내용, 신뢰도)
4. **검증 기준**: 소스 신뢰도 판단 기준

### Step 3: 정보 수집 및 정리

`WebSearch`와 기타 도구로 정보를 수집하되, 수집 즉시 구조화한다:

```
[소스 ID: S-01]
  출처: [기관/저자명]
  URL: [URL]
  발행일: [YYYY-MM-DD]
  신뢰도: [상/중/하] — [판단 근거]
  관련 RQ: [RQ-1, RQ-3]
  핵심 내용:
    - [요약 1]
    - [요약 2]
  인용 가능 구절: "..."
```

소스 신뢰도 판단 기준:
- **상**: 공식 기관, 피어리뷰 논문, 공인 통계
- **중**: 전문 미디어, 산업 보고서, 전문가 블로그
- **하**: 개인 블로그, 익명 게시글, 출처 불명확

### Step 4: 종합 및 보고서 작성

수집된 정보를 RQ별로 종합한다:

```
[RQ-1 답변]
  핵심 결론: (두괄식)
  근거 1: [S-01] ...
  근거 2: [S-03] ...
  반대 의견: [S-05] ...
  종합 판단: ...
```

보고서 구조:
1. 서론 — 리서치 배경, RQ 목록, 방법론
2. RQ별 분석 — 각 질문에 대한 구조화된 답변
3. 종합 비교 — 교차 분석, 비교표, 관계도
4. 결론 — 핵심 인사이트, 한계점
5. 참고문헌 — 전체 소스 리스트

### Step 5: Lint 검증

리서치 완료 후, 아래 5개 규칙을 순서대로 검증한다.

| ID | 규칙 | 기준 |
|----|------|------|
| R-01 | 출처 존재 | 모든 사실 주장에 출처(URL/논문/날짜) 존재 |
| R-02 | 발행일 범위 | 출처 발행일이 요구된 기간 내 |
| R-03 | RQ 완결성 | 모든 RQ에 대한 답변이 보고서에 포함 |
| R-04 | 소스 다양성 | 단일 소스 의존도 ≤ 40%, 최소 3개 독립 소스 |
| R-05 | 관점 균형 | 상반된 관점이 공정하게 다뤄짐 |

## Lint 검증 보고 형식

```
━━━ Research Lint Report ━━━
✅ R-01: 출처 존재 — 사실 주장 20건 중 20건 출처 확인
⚠️ R-02: 발행일 — 소스 2개가 기간(2024~2025) 외 → 보완 소스 추가
✅ R-03: RQ 완결성 — 4개 RQ 모두 답변 포함
❌ R-04: 소스 다양성 — S-03이 전체 인용의 45% → 추가 소스 보강
✅ R-05: 관점 균형 — 주요 결론에 대한 반론 3건 포함

결과: 2건 교정 완료, 재검증 통과 ✅
━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

## 주의사항

- 출처 없는 주장은 절대 포함하지 않는다. "일반적으로 알려진 바에 따르면"은 허용하지 않는다.
- 최신성이 중요한 주제(기술, 시장)는 2년 이내 자료를 우선한다.
- 검색 결과가 부족하면 사용자에게 투명하게 알린다. 없는 정보를 만들어내지 않는다.
- 번역/요약 시 원문의 의미를 왜곡하지 않는다. 불확실하면 원문을 함께 제시한다.
