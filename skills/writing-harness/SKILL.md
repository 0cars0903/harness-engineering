# Writing Harness — 글쓰기 전용 하네스

모든 글쓰기 작업에서 일관된 품질을 보장하는 하네스 스킬이다.
Blueprint로 구조를 잡고, Lint Rules로 품질을 강제한다.

---

## 워크플로우

### Step 1: 글쓰기 컨텍스트 파악

다음을 확인한다 (대화에서 이미 파악된 내용은 건너뛴다):

1. **문서 유형**: 자소서/보고서/블로그/이메일/기획서 등
2. **독자**: 누가 읽는가? (배경지식 수준, 관심사, 기대)
3. **목적**: 설득/정보 전달/감동/교육/보고 중 무엇인가?
4. **분량**: 글자 수 또는 페이지 수 제한
5. **톤앤매너**: 격식/비격식, 진지/경쾌 등
6. **SSOT**: 사실 정보의 출처 (파일, URL, 사용자 구술)

### Step 2: Blueprint 설계

문서 유형에 따라 구조를 설계한다.

**두괄식 구조 (기본):**
모든 섹션의 첫 문장은 그 섹션의 핵심 결론이다. 독자가 첫 문장만 읽어도 요지를 파악할 수 있게 한다.

```
[섹션 1: 핵심 메시지]
  첫 문장: 이 글에서 전달하고자 하는 핵심 결론
  지지 근거 1~3개

[섹션 2: 맥락/배경]
  첫 문장: 이 배경이 중요한 이유
  구체적 상황 설명

[섹션 3: 본론/전개]
  첫 문장: 이 섹션의 핵심 주장
  근거, 사례, 데이터

[섹션 N: 결론/제안]
  첫 문장: 최종 결론
  다음 행동 제안
```

**분량 배분**: 각 섹션의 비율을 명시한다. 예: 도입 10%, 본론 70%, 결론 20%.

### Step 3: 초안 작성

Blueprint에 따라 초안을 작성한다.

작성 원칙:
- 한 문장 = 한 메시지. 복합문 최소화.
- 국문 문장 길이 60자 이내 지향.
- 수동태보다 능동태. 추상명사보다 동사.
- 숫자는 반드시 단위와 함께 ("30%" O, "30" X).
- 금지 어휘 회피: "다양한", "여러 가지", "많은 것들", "기여하고 싶습니다", "열정을 가지고".

### Step 4: Lint 검증

초안 완성 후, 아래 8개 규칙을 순서대로 검증한다.

**위반 시 자동 교정을 시도하고, 교정 결과를 사용자에게 보고한다.**
Lint를 통과하지 못한 결과물은 최종 출력하지 않는다.

| ID | 규칙 | 기준 |
|----|------|------|
| W-01 | 문장 길이 | ≤ 60자 (국문) / ≤ 25단어 (영문) |
| W-02 | 수동태 비율 | ≤ 15% |
| W-03 | 헤딩 계층 | 최대 H3까지 |
| W-04 | 단락 길이 | 한 단락 최대 5문장 |
| W-05 | 금지 어휘 | 리스트 대조 (references/lint-rules.md 참조) |
| W-06 | 두괄식 | 각 섹션 첫 문장 = 핵심 결론 |
| W-07 | 숫자 단위 | 모든 숫자 뒤에 단위 존재 |
| W-08 | SSOT 정합성 | 사실 정보가 SSOT와 일치 |

상세 규칙은 `references/lint-rules.md`를 참조한다.

### Step 5: 최종 출력

Lint를 모두 통과하면, 적절한 형식으로 최종 출력한다:
- 문서 파일이 필요한 경우 → `docx` 스킬로 .docx 생성
- 마크다운으로 충분한 경우 → .md 파일 생성
- 대화 내 표시로 충분한 경우 → 그대로 표시

## Lint 검증 보고 형식

```
━━━ Writing Lint Report ━━━
✅ W-01: 문장 길이 — 전체 42문장 중 위반 0개
⚠️ W-02: 수동태 비율 — 18% (기준: 15%) → 3문장 능동태로 교정
✅ W-03: 헤딩 계층 — H3 이내
✅ W-04: 단락 길이 — 전체 8단락 중 위반 0개
❌ W-05: 금지 어휘 — "다양한" 2회 발견 → 구체적 표현으로 교체
✅ W-06: 두괄식 — 전체 4섹션 첫 문장 결론 확인
✅ W-07: 숫자 단위 — 전체 숫자 7개 중 위반 0개
✅ W-08: SSOT 정합성 — 사실 정보 12건 검증 완료

결과: 2건 교정 완료, 재검증 통과 ✅
━━━━━━━━━━━━━━━━━━━━━━━━━
```
