# Analysis Harness — 기획/분석 전용 하네스

비즈니스 분석과 기획 작업에서 논리적 일관성과 데이터 기반 의사결정을 보장하는 하네스다.

---

## 워크플로우

### Step 1: 분석 컨텍스트 파악

다음을 확인한다:

1. **분석 목적**: 무엇을 결정하기 위한 분석인가?
2. **의사결정자**: 누가 이 분석을 보고 결정을 내리는가?
3. **분석 범위**: 시간 범위, 지리적 범위, 산업 범위
4. **데이터 소스**: 활용 가능한 데이터와 접근 방법
5. **결과물 형식**: 보고서/프레젠테이션/대시보드/요약문

### Step 2: 프레임워크 선정 및 가설 수립

분석 목적에 맞는 프레임워크를 선정한다:

| 분석 목적 | 권장 프레임워크 |
|-----------|---------------|
| 시장 환경 이해 | Porter's Five Forces, PESTLE |
| 내부 역량 평가 | SWOT, Value Chain Analysis |
| 경쟁 비교 | 벤치마킹 매트릭스, 포지셔닝 맵 |
| 전략 방향 | Ansoff Matrix, BCG Matrix |
| 우선순위 결정 | 의사결정 매트릭스, Impact-Effort Grid |
| 원인 진단 | Fishbone (이시카와), 5 Whys, Pareto |
| 재무 판단 | 비용-편익 분석, NPV, IRR |

프레임워크 선정 후, 검증 가능한 가설을 수립한다:
- 가설은 "X이면 Y이다" 형식으로 명확히 기술
- 각 가설에 검증 방법과 필요 데이터를 매핑

### Step 3: Blueprint 설계

분석 보고서의 구조를 미리 설계한다:

```
[Executive Summary] — 핵심 결론 + 권고사항 (1페이지)
[분석 배경] — Context + Problem 정의
[분석 방법론] — 사용 프레임워크 + 데이터 소스
[본문 분석] — 프레임워크별 분석 결과
[종합 비교] — 비교 매트릭스 또는 종합 평가
[결론] — 가설 검증 결과
[제안] — Actionable 권고사항 (Who/What/When/How)
```

시각화 계획도 Blueprint에 포함한다:
- 어떤 차트를 어디에 배치할지
- 각 차트가 전달하는 메시지는 무엇인지

### Step 4: 분석 실행

Blueprint에 따라 분석을 수행한다:
- 데이터 수집 → `xlsx` 스킬, `WebSearch`, DB 조회
- 프레임워크 적용 → 구조화된 분석
- 시각화 → 차트, 표, 매트릭스 생성
- 인사이트 도출 → "So What?" 테스트

**"So What?" 테스트**: 모든 분석 결과에 "그래서 어떻다는 건가?"라고 질문한다. 답변할 수 없으면 인사이트가 부족한 것이다.

### Step 5: Lint 검증

분석 완료 후, 아래 5개 규칙을 순서대로 검증한다.

| ID | 규칙 | 기준 |
|----|------|------|
| A-01 | 출처 명시 | 모든 데이터 주장에 출처 존재 |
| A-02 | MECE 충족 | 분석 범주가 상호 배타적 + 전체 포괄적 |
| A-03 | 논리적 도출 | 결론이 근거에서 추론 가능 |
| A-04 | 가설-검증 매핑 | 모든 가설에 검증 결과 1:1 매핑 |
| A-05 | Actionable 제안 | 실행 가능한 권고 포함 (Who/What/When/How) |

## Lint 검증 보고 형식

```
━━━ Analysis Lint Report ━━━
✅ A-01: 출처 명시 — 데이터 주장 15건 중 15건 출처 확인
⚠️ A-02: MECE — 분석 범주 6개 중 1개 중복 가능성 → 범주 재조정
✅ A-03: 논리적 도출 — 결론 3개 모두 근거 기반 확인
✅ A-04: 가설-검증 — 가설 4개 중 4개 검증 완료
❌ A-05: Actionable — 제안 2번에 실행 주체(Who) 누락 → 보완

결과: 2건 교정 완료, 재검증 통과 ✅
━━━━━━━━━━━━━━━━━━━━━━━━━━
```

## 주의사항

- 상관관계와 인과관계를 혼동하지 않는다. "A가 증가하면서 B도 증가했다"는 상관이지 인과가 아니다.
- 확증 편향을 경계한다. 가설을 지지하는 데이터만 선별하지 않도록 반증 자료도 적극 탐색한다.
- 분석의 한계를 명시한다. 데이터 부족, 시간 제약, 가정 등을 투명하게 기술한다.
- 의사결정자의 시간을 존중한다. Executive Summary에서 30초 안에 핵심을 전달할 수 있게 한다.
